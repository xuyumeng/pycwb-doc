pycwb.search module¶

pycwb.search.analyze_job_segment(config, job_seg)[source]¶

Analyze one job segment with the given configuration

This function includes the following stages:

  1. Read data from job segment (pycwb.modules.read_data.read_from_job_segment)

  2. Data conditioning (pycwb.modules.data_conditioning.data_conditioning)

  3. Create network (pycwb.modules.coherence.create_network)

  4. Coherence (pycwb.modules.coherence.coherence)

  5. Supercluster (pycwb.modules.super_cluster.supercluster)

  6. Likelihood (pycwb.modules.likelihood.likelihood)

The results will be saved to the output directory in json format on likelihood stage

Parameters:

  • config (Config) – configuration

  • job_seg (WaveSegment) – job segment

pycwb.search.search(user_parameters='./user_parameters.yaml', log_file=None, log_level='INFO', no_subprocess=False)[source]¶

Main function to run the search

This function will read the user parameters, select the job segments, create the catalog, copy the html and css files and run the search in subprocesses by default to avoid memory leak.

Parameters:

  • user_parameters (str) – path to user parameters file

  • log_file (str, optional) – path to log file, defaults to None

  • log_level (str, optional) – log level, defaults to ‘INFO’

  • no_subprocess (bool, optional) – run the search in the main process, defaults to False (Set to True for macOS development)
